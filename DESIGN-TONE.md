# DESIGN-TONE.md — 前端审美基调(给前端任务的设计锚点)

> 前端取向(已定):**功能尽量复刻 Partiful;视觉不逐像素抄 Partiful,而是按本文件的基调做一套自洽、有质感的 UI + 适度动效。**
> 本文件是设计锚点,不是死规定。动手前**先读 `frontend-design` SKILL.md**,按其"先出 token 系统 + 自我批判"的流程做。
> **铁律(不随审美自由而打折)**:RSVP 前后分级的真实呈现、guest_token 存取、私密读取走 SSR、数据正确呈现 —— 审美可自由,这些语义必须正确。

## 这个产品是什么(给设计 pin 一个主体)

一个让年轻华人群体(参考 JU 那批用户)组织线下小聚会的活动邀请平台。单页的核心工作:**让收到链接的人在 10 秒内看懂"什么活动、什么时候、谁来了",并轻松点下 RSVP**。社交、轻松、有当下感,不是企业工具的严肃感,也不要婚礼请柬的繁复。

## 基调方向(锚点,非强制)

- **气质**:干净、现代、社交感、移动优先。轻盈而非厚重;有呼吸感的留白;克制的微动效服务于"活动正在发生"的当下感。
- **避免**:① 模板默认款(Bootstrap/未改造的 shadcn 原样);② frontend-design 点名的三种 AI 默认聚类(暖米色+高对比衬线+陶土色 / 近黑底+单一荧光色 / 报纸式细线密排)。这三种是"默认"不是"选择",除非有真实理由别用。
- **暗色友好**:社交场景夜间使用多,优先保证暗色下舒适(不必强制只做暗色,但暗色不能是事后补丁)。
- **动效**:遵循 frontend-design —— 一个编排得当的时刻胜过满屏零散特效。候选场景:活动页载入序列、RSVP 成功的反馈、名单实时新增的入场。**reduced-motion 必须尊重**。主题"特效"(theme/effect 字段)给一个像样实现即可,不堆砌。

## 落地要求

- 按 frontend-design 流程:先在思考中出 4–6 色 token + 2+ 字体角色(display/body/utility)+ 布局概念 + 一个"签名元素",自我批判过一遍确认不是默认款,再写代码。
- **签名元素**:把"boldness 花在一个地方"。建议落在活动页的 hero(海报+标题+日期+人数的组合)或 RSVP 交互上 —— 这是用户记住产品的地方。其余保持安静克制。
- **文案**(按 frontend-design 写作原则):主动语态、句子大小写、按用户认知命名("加入日历"而非"导出 .ics")、动作全流程同名("RSVP"按钮→"已回复"提示)、空/错态给方向不卖惨。
- **质量地板**(不声张但必须有):响应式到移动端、可见键盘焦点、reduced-motion 尊重、对比度达标。

## 与安全语义的交界(前端必须正确呈现的)

- 未 RSVP 的活动页**真实地**不渲染完整地址和名单(不是 CSS 隐藏 —— 数据层就不该返回;前端也不得从任何地方拼出地址)。
- guest_token 存 localStorage,**绝不出现在 URL 或任何可分享处**。
- 私密活动页只走 SSR(经受信角色),不在客户端直连 private RPC。
- 人数显示精确数字(除非 host 隐藏);Can't Go 名单永不显示给访客。
