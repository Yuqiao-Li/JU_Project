# PRD: 局详情 event-page（访客侧 · `/[slug]`）—— 局卡中心化

> 见 [README](./README.md)。本轮只定产品功能范围，不含实现。新功能改本文件。

## 目的
访客（多经扫码/链接进入）看到局、留个位置、拿到个人化票根；成局后双向解锁联系方式。

## 当前功能（已建）
海报 hero（主题色/特效）· 分级解锁（城市→RSVP 后地址）· RSVP（姓名+可选+1+contact+微信 R4）· 候补 ·
密码门 · 名单(Going/Maybe) · 评论流 · 日期投票 · 加入日历 · 取消/结束/锁定横幅。

## MVP 需要的功能（局卡中心化重构）
- **结构**：**局卡（态2 实时进度）在顶 + 留位表单在下**。留位表单填 姓名 / 微信（必填）+ 通用联系方式 / 一句话自我介绍（可选）+ 是否能来（going/maybe/not）。
- **保存后整页变个人化局卡**（态2：昵称 + 你的状态 + 成局进度）→ **点开可展开编辑**个人信息。
- **保留**：成局后地址解锁；取消/结束/锁定横幅；密码门。

## 缓做 / 不做（从 MVP 局详情移出，代码保留）
名单（§6.4 隐藏参与者）· 评论流 · 日期投票。

## 依赖的成局契约
留位必填/可选、双盲（成局前 host 见姓名+介绍、联系方式成局后双向解锁、访客互不见）、成局进度 —— 见 [README 跨页契约](./README.md#跨页成局契约)。局卡交互见 [event-card](./event-card.md)。
