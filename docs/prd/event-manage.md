# PRD: 管理 event-manage（`/dashboard/events/[id]`）—— 局卡中心化

> 见 [README](./README.md)。本轮只定产品功能范围，不含实现。新功能改本文件。

## 目的
host 管理一个局：看谁留位、确认成局、看成局后联系方式、改局、生命周期。

## 当前功能（已建）
人数卡 · 名单(含 contact) · 复制全部联系方式 · 候补提升 · 生命周期(发布/下架/取消/删除) ·
R4 锁定（两步/不可逆）· 锁定后双向微信（阅后即焚）· 公开链接/以访客预览。

## MVP 需要的功能（局卡中心化）
- **管理台页 = 该局卡 + host 管理表单（态2 点开展开）**，与访客侧"点开编辑"同构。
- **管理面**含：host 可见名单（姓名 + 自我介绍）/ **确认成局** / 编辑局 / 生命周期 / **成局后联系方式（双向）**。
- 凑满 → 给 host"可成局了，确认？"提示；到时自动锁；host 可随时手动锁（R4 已有）。

## 缓做 / 不做
host 静默拉黑（V2）· 出席邮戳/票根墙（V2）· 逐人审核（不做，整体确认）。

## 依赖的成局契约
成局触发与确认（整体）、成局前可见姓名+介绍、成局后双向联系、静默隐藏 —— 见 [README 跨页契约](./README.md#跨页成局契约)。局卡交互见 [event-card](./event-card.md)。
