### Interim 品牌表述（PDF 到位前的参考；源 `JU 细节纠偏.md` §1-7 + `DESIGN-TONE.md`）
- **三王牌**：
  - **纯粹组局** —— 基本单位是"局"不是"人"；不做内容表演 / 个人广场 / 粉丝体系 / 站内长期群聊；直奔线下见面，用完即走。
  - **无压力组局** —— 报名像"表达兴趣"而非提交审核表单；开局像随口喊人；**绝不让用户被逐个挑选 / 公开社死 / 看到组局失败**。
  - **懂你的活动** —— 北美华人留学生 + 职场新人；高共鸣黑话分类对冲新城市孤独；不贩卖社交焦虑。
- **气质**：**轻而靠谱**；**工具感、不做广场**；**入口是局不是人**（§5）；**双盲隐私**（联系方式成局后才双向开放、阅后即焚；隐藏参与者名单；女性安全感）。
- **受众**：北美华人 23–35（留学生 / 刚搬家 / 职场新人）。Venmo 等本地支付 OK；微信是联系方式主轴。
- **视觉基调**：见 `DESIGN-TONE.md`（干净 / 现代 / 移动优先 / 暗色友好 / 克制微动效；"boldness 花在一个地方"= 局卡/海报）。

## 三、要审查的功能模块（对照品牌）
完整逐页见 `docs/prd/`（每页 PRD + `README.md` 的成局逻辑 + §路线 + 各页"实现逻辑"）。当前功能版图：
- **成局闭环**：开局(缺几人/分类) → 看见(局卡) → 留位(姓名/微信必填，通用联系/一句话介绍可选) → 等确认(host 成局前见姓名+介绍) → 成局(整体确认；满→提示 host / 到时自动锁 / 手动锁) → 双向联系方式(双盲、成局后、阅后即焚) → 会出现(轻提示加群) → 未成局静默隐藏 → 一键复用。
- **局卡**：可分享的多状态数码票根（态1 艺术+二维码 / 态2 个人化进度；扫码入局）。
- **页面**：落地 `/`、发现 `/discover`、活动页 `/[slug]`(局卡中心化)、管理台 `/[id]`(局卡中心化)、仪表盘 `/dashboard`(局卡顶+复用)、设置(单昵称+微信+通用联系方式)、登录。
- **已删**：`/u/` 主办主页（§5 入口是局不是人；RPC/列保留闲置）。
- **🟡 明确不做（V1）**：cohost / 相册 / 嘉宾审核 / 问卷 / 评论反应 / 提醒 / 广播 / 匿名名单 / lat-lng。
- **V2 / 缓做**（见 `docs/prd/README.md` §路线）：倒计时 / 出席邮戳 / 静默拉黑 / 邮箱认证 / Vibe Filter / 城市-邻里过滤 / 首页局卡流 等。
